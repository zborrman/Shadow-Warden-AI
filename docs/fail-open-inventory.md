## Review policy

1. Payment / billing / auth fail-opens must emit a fixed-schema audit
   log line (`payment_bypassed` pattern from v7.4) — verify per site.
2. Filter-pipeline fail-opens (stage skipped on error) must increment a
   Prometheus counter so silent degradation is alertable.
3. Anything not in the protected table needs an explicit decision:
   keep-open (document why), flag-gated fail-closed, or hard fail-closed.
