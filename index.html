Core Protection Technologies

SecretRedactor

Automatically strips API keys, SSNs, IBANs, and credit card numbers using 15 battle-tested regex patterns — before a single token leaves your perimeter.

SemanticGuard

Dual-layer detection: regex rule engine plus all-MiniLM-L6-v2 cosine similarity — catches jailbreaks that keyword filters miss entirely.

ObfuscationDecoder

Pre-filter that decodes Base64, hex, ROT13, and Unicode homoglyph attacks before they reach the detection pipeline.

EvolutionEngine

Auto-generates new detection rules via Claude Opus streaming when novel attack patterns are detected — hot-reloads without a restart.

Analytics & Observability

Real-time Streamlit security dashboard, Prometheus metrics, Grafana alerts, and Splunk/Elastic SIEM integration — zero PII ever logged.

Drop-in Compatibility

OpenAI-compatible proxy endpoint — point OPENAI_BASE_URL at Warden and keep your existing SDK code unchanged. LangChain callback included.

GDPR by Design

Content is never stored — only metadata (type, length, timing). Built-in /gdpr/export and /gdpr/purge endpoints for Article 17 compliance.

Pipeline Architecture

POST /filter
  │
  ├─ Stage 0a  Redis content-hash cache          (5-min TTL, fail-open)
  ├─ Stage 0b  ObfuscationDecoder               (base64 · hex · ROT13 · homoglyphs)
  ├─ Stage 1   SecretRedactor                   (15 PII/secret patterns)
  ├─ Stage 2   SemanticGuard (rules)            (compound risk escalation)
  ├─ Stage 2b  SemanticBrain (MiniLM ML)        (cosine similarity, async)
  └─ Decision  → ALLOW / HIGH / BLOCK
                │
                └─ HIGH/BLOCK → EvolutionEngine (Claude Opus, background)
                                       │
                                       └─ add_examples() hot-reload → corpus
