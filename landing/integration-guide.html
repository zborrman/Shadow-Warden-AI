9-Layer Defense Architecture

Every request passes through all 9 layers sequentially. Layers 1–3 are pre-semantic (fast, <2ms each). Layers 4–7 are semantic & ML. Layers 8–9 are decision & learning.

1
Topological Gatekeeper (TDA)
N-gram point cloud → β₀/β₁ Betti numbers. Detects structural noise before any semantic analysis. Adaptive threshold: code 0.65 / text 0.82.
<2ms
2
Obfuscation Decoder
Depth-3 recursive decode: Base64, Hex, ROT13, Caesar, word-split, UUencode, unicode homoglyphs. Strips all encoding layers before analysis.
<1ms
3
Secret Redactor
15 regex patterns (API keys, JWT, SSH, credit cards, SSN) + Shannon entropy scan for unknown secrets. GDPR: content never logged.
<1ms
4
ThreatVault Semantic Rules
1,300+ curated threat signatures. Compound risk escalation: 3+ MEDIUM signals → HIGH. Pattern categories: injection, exfiltration, roleplay bypass, tool abuse.
<2ms
5
Hyperbolic Brain (MiniLM)
MiniLM sentence embeddings projected onto Poincaré ball. Final score = 70% cosine + 30% hyperbolic distance. Adversarial suffix stripping pre-processing.
4–6ms
6
NeMo Bridge (v2.9)
GeometricThreatBridge translates threat coordinates into Colang context variables. Routes to 5 Colang flows: clean / cautious / restricted / obfuscation / blocked.
<1ms
7
Causal Arbiter
Bayesian DAG with Pearl do-calculus. Gray-zone escalation only. 5 causal nodes, backdoor correction. P(HIGH_RISK|evidence) threshold: 0.65.
1–5ms
8
ERS + Shadow Ban Engine
Redis sliding window. Differentiated strategies: gaslight (prompt injection), delay 3s (bot/stuffing), standard. Shadow ban activates at score ≥ 0.75.
<1ms
9
Decision + Evolution Engine
Final ALLOW / BLOCK / SHADOW_BAN. HIGH/BLOCK events trigger Claude Opus background learning. Hot-reload new rules without restart. Evidence Vault → MinIO.
async

Hybrid Server Architecture Guide

Shadow Warden is built on a Data-Gravity principle: all sensitive data and security metadata stays inside the client's infrastructure. Only clean, anonymized tokens can flow to cloud LLM providers.

🏢 On-Premises / Private Cloud

  • Shadow Warden gateway (port 8001)
  • MinIO object store (evidence vault)
  • PostgreSQL (audit logs)
  • Redis (ERS sliding window)
  • Prometheus + Grafana
  • MiniLM model weights
  • Raw prompts (never leave)
  • PII / PHI / secrets (redacted here)
  • Evidence bundles (SHA-256 signed)
↔ Filtered
clean tokens
only
↔ mTLS / TLS 1.3

☁️ Cloud LLM Providers

  • OpenAI / Azure OpenAI
  • Anthropic Claude
  • Google Gemini / Vertex AI
  • AWS Bedrock
  • NVIDIA NeMo (optional)
⚠ Zero PII/PHI/secrets ever reach cloud providers. Masking Engine tokens are unmapped on-prem before any cloud call.

Deployment Topology Options

TopologyDescriptionBest ForGDPR
1. Full On-Prem All 11 Docker services run on client hardware. No cloud egress at all. Air-gapped mode available. Gov, Defense, Healthcare (EU) ✅ Full Art.35
2. Hybrid (recommended) Gateway + storage on-prem. Cloud LLM only receives clean tokens through Warden proxy. Enterprise, Financial, SaaS ✅ Compliant
3. Multi-Region Hybrid One Warden instance per region (EU/US). Each MinIO stores only local-origin evidence. Cross-region telemetry aggregated in Prometheus. Global Enterprise, FinServ ✅ Data residency preserved
4. Kubernetes (Production Scale) Warden as sidecar or dedicated gateway pod. HPA on CPU. Redis Cluster for ERS at scale. High-volume SaaS, MLOps ✅ With proper PVC config

NVIDIA NeMo Guardrails Bridge (v2.9)

The GeometricThreatBridge is the world's first integration that translates mathematical threat geometry — Poincaré ball hyperbolic distances, TDA Betti numbers, and Bayesian causal probabilities — into Colang symbolic flow variables.

What GeometricThreatBridge Does

Shadow Warden computes threat coordinates in multiple mathematical spaces. The bridge maps these to Colang context variables that route NeMo's conversational logic.

CoordinateColang VariableRange
Hyperbolic distance (Poincaré)$hyperbolic_risk0.0–1.0
β₀/β₁ Betti score (TDA)$topology_noise0.0–1.0
P(HIGH_RISK) Bayesian DAG$causal_p_risk0.0–1.0
ERS sliding window score$entity_risk0.0–1.0
Resolved threat class$threat_classenum

5 Colang Flow Routes

clean_flow — Standard LLM response. No restrictions.
cautious_flow — Soft-refusal on sensitive topics. Gray zone.
restricted_flow — High entity risk. Hard content guardrails.
obfuscation_flow — Decoded obfuscation detected. Specific response.
blocked_flow — BLOCK decision. No LLM call made.

Integration Code

from warden.integrations.nemo_bridge import GeometricThreatBridge, create_nemo_integration
from nemoguardrails import RailsConfig, LLMRails

bridge, synthesizer = create_nemo_integration(
    colang_dir="warden/integrations/nemo_colang",
    evolution_dir="warden/integrations/nemo_colang/generated"
)

config = RailsConfig.from_path("warden/integrations/nemo_colang")
rails = LLMRails(config)

# Per-request: translate Warden result → Colang context
filter_result = await warden_client.filter_async(user_message)
colang_context = bridge.to_colang_context(filter_result)

# NeMo routes through the correct Colang flow automatically
response = await rails.generate_async(
    messages=[{"role": "user", "content": user_message}],
    context=colang_context
)

Obsidian Business Community Integration (v4.11)

The Shadow Warden Obsidian plugin bridges private knowledge management with enterprise-grade AI security. Notes are scanned locally before any AI interaction or community share.

5 REST Endpoints

EndpointPurpose
POST /obsidian/scanScan note — returns data_class, secrets_found, risk_score
POST /obsidian/shareShare to community — issues SEP UECIID, blocks if secrets found
GET /obsidian/feedCommunity shared notes feed (paginated)
POST /obsidian/ai-filterPre-filter note content through full /filter pipeline before AI
GET /obsidian/statsVault scan statistics per tenant

Data Classification Logic

Resolution order for each note:

  1. Explicit: YAML frontmatter data_class: PHI
  2. Tags: #phi, #pii, #financial, #classified
  3. Keywords: regex match (SSN, patient, credit card, IBAN…)
  4. Default: GENERAL
Notes with data_class: CLASSIFIED or secrets_found: true are blocked from share — the SEP UECIID is never issued.
