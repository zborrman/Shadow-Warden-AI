Benchmark Methodology

Transparent, reproducible, open to third-party verification

Corpus Design

44 hand-labelled samples: 32 attacks across 7 categories (prompt injection, jailbreak, obfuscation, data exfiltration, secret leakage, PII, tool abuse) + 12 benign controls. Designed to reflect real-world production traffic patterns.

Evaluation Protocol

Binary classification: allowed=false ≡ detected. F1 = 2·P·R/(P+R). Macro-average across categories. No train/test leakage — corpus is held-out from the corpus used to tune SEMANTIC_THRESHOLD.

Latency Measurement

200 warm-up requests (discarded) + 200 timed requests via in-process TestClient (eliminates network jitter). P-values computed with Python statistics.quantiles. Hardware: 4 vCPU / 8 GB RAM, CPU-only torch.

Reproduction

Full corpus and harness are open: scripts/benchmark.py in the public repo. Run locally with no API key or running stack needed. Export: --export results.json for JSON output, --markdown for table.

OWASP Coverage

Maps to OWASP LLM Top 10 2025: LLM01 (Prompt Injection), LLM02 (Insecure Output Handling / data exfiltration), LLM06 (Sensitive Information Disclosure). Agent abuse patterns added for LLM08.

Version Policy

Results are version-stamped. Re-run after every minor version bump. Results files committed to results/benchmark_YYYYMMDD.json. Historic results retained for regression tracking.
