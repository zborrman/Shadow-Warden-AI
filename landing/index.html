How It Works

9-Layer Defense Pipeline

Every request runs through all layers sequentially. Any layer can block. End-to-end P99 <19ms on CPU-only hardware.

Stage 1 — Topological Gatekeeper
Topological Data Analysis pre-filter runs in <2ms before any ML or regex. Converts text into an n-gram point cloud and computes β₀/β₁ Betti numbers to detect structural noise characteristic of obfuscated payloads.
  • N-gram point cloud construction
  • β₀ connected components · β₁ 1-cycles
  • Noise score: 0.33×char + 0.27×wc + 0.22×diversity + 0.10×β₀ + 0.08×β₁
  • Adaptive thresholds: code (0.65) vs natural language (0.82)
  • Ripser optional — algebraic fallback for CPU-only

Attack vectors

«Ignore previous instructions» overrides
Indirect injection via RAG documents or tool results
Base64 / ROT13 / Unicode obfuscation to bypass keyword filters

LLM02
Sensitive Information Disclosure
HIGH

LLMs may inadvertently expose API keys, PII, confidential data, or training-set memorization in their responses. Attackers probe models to extract secrets embedded in prompts or memorised during training.

Attack vectors
API keys, tokens, and credentials in user prompts forwarded to LLM
Training data extraction via repetition attacks («repeat the words above…»)
PII — names, emails, phone numbers, credit cards — leaked in context
Mass credential harvesting via bulk queries

GDPR: content never logged — only metadata (type, length, timing)
Actively protected

SQL injection via AI-generated database queries (UNION SELECT, DROP TABLE)
OS command injection — shell metacharacters in AI-generated scripts

Path traversal — ../../etc/passwd in AI-generated file paths
