Compliance

Full OWASP LLM Top 10 Coverage

Shadow Warden maps directly to every risk category. Click any card for full threat detail and how we neutralise it.

LLM01
Prompt Injection
CRITICAL

Adversarial input manipulates an LLM into overriding its instructions or taking unintended actions. Includes direct injection (user crafts the payload) and indirect injection (payload hidden in documents, URLs, or tool outputs the model processes).

Attack vectors
DAN / STAN / AIM / DUDE persona jailbreaks
«Ignore previous instructions» overrides
Indirect injection via RAG documents or tool results
Base64 / ROT13 / Unicode obfuscation to bypass keyword filters
Fictional, hypothetical, and roleplay framing
Shadow Warden protection
ThreatVault Stage 1.5 — 50+ jailbreak signatures matched before ML
SemanticBrain (MiniLM) cosine similarity against adversarial corpus
ObfuscationDecoder strips base64 / hex / ROT13 / homoglyphs at Stage 0b
EvolutionEngine auto-generates new rules from blocked attempts (Claude Opus)
Actively protected
LLM02
Sensitive Information Disclosure
HIGH

LLMs may inadvertently expose API keys, PII, confidential data, or training-set memorization in their responses. Attackers probe models to extract secrets embedded in prompts or memorised during training.

Attack vectors
API keys, tokens, and credentials in user prompts forwarded to LLM
Training data extraction via repetition attacks («repeat the words above…»)
PII — names, emails, phone numbers, credit cards — leaked in context
Mass credential harvesting via bulk queries
Shadow Warden protection
SecretRedactor — 15+ regex patterns: API keys, JWT, credit cards, SSNs, IBANs
Redaction policies: FULL (replace), MASKED (last 4 chars), RAW (detect-only)
PII Yellow-Zone masking — vault tokenisation with reversible session IDs
GDPR: content never logged — only metadata (type, length, timing)
Actively protected
LLM03
Supply Chain Vulnerabilities
HIGH

Compromised third-party model providers, plugins, datasets, or pre-trained weights introduce backdoors or malicious behaviour before the application even runs. Attackers target the model supply chain rather than the deployed system.

Attack vectors
Poisoned Hugging Face or PyPI packages with trojanised weights
Malicious fine-tuning datasets injected via public contribution pipelines
Compromised plugins or tool integrations in LLM agent frameworks
Dependency confusion / typosquatting in ML library installs
Shadow Warden protection
Pinned dependencies (requirements-lock.txt) — reproducible, auditable builds
Dockerfile builds from verified MCR base images — no public Hugging Face at runtime
All inter-service communication is internal Docker network — no external model calls
CI/CD pipeline validates Docker build integrity on every commit
Actively monitored
LLM04
Data & Model Poisoning
CRITICAL

Malicious examples are injected into the model's training data or in-context learning corpus to install hidden backdoors, alter behaviour on trigger inputs, or degrade detection accuracy for specific attack classes.

Attack vectors
Sleeper agent — trigger word activates hidden behaviour after fine-tuning
Few-shot corpus injection — adversarial examples flood the in-context corpus
Gradient-based backdoor insertion into model weights
Persistent session override — «from now on ignore all restrictions»
Shadow Warden protection
DataPoisoningGuard (Stage 2c) — statistical anomaly detection on corpus growth
Corpus cap (500 examples) + vetting — auto-rejects examples with ≥10% similarity
EvolutionEngine vets all Claude-generated rules before corpus insertion
ThreatVault signature: sleeper agent trigger patterns blocked at Stage 1.5
Actively protected
LLM05
Insecure Output Handling
HIGH

Unsanitised LLM output is passed to downstream components — browsers, shells, databases — causing XSS, command injection, SQL injection, SSRF, or path traversal. The LLM becomes the attack vector into the rest of the stack.

Attack vectors
XSS — <script> tags or event handlers in AI-rendered HTML
SQL injection via AI-generated database queries (UNION SELECT, DROP TABLE)
OS command injection — shell metacharacters in AI-generated scripts
SSRF — AI instructs fetch to cloud metadata endpoint (169.254.169.254)
Path traversal — ../../etc/passwd in AI-generated file paths
Shadow Warden protection
OutputGuard — scans all AI-generated text before it reaches downstream systems
SemanticGuard LLM05 rules — XSS, SQLi, CMDi, SSTI, XXE, SSRF, path traversal patterns
ThreatVault — dedicated signatures for each injection class
/output/scan endpoint — standalone output scanner for existing pipelines
Actively protected
LLM06
Excessive Agency
CRITICAL

An LLM agent is granted excessive permissions or takes high-impact autonomous actions without human approval. Attackers instruct the agent to delete files, exfiltrate data, make API calls, or modify infrastructure on their behalf.

Attack vectors
Mass deletion — «delete all database records» via injected instruction
Data exfiltration — agent POSTs sensitive data to attacker-controlled webhook
Privilege escalation — agent modifies ACLs or IAM roles autonomously
Supply chain abuse — agent installs malicious packages via npm/pip
Shadow Warden protection
AP2 Agentic Payment Protocol — 7-layer mandate validation before any agent action
WalletShield — hard token budget limits; blocks runaway cost from agent loops
SemanticGuard LLM06 rules — flags destructive bulk operations before execution
AgentMonitor — tracks all agent calls and kills sessions exceeding risk threshold
Actively protected
LLM07
System Prompt Leakage
HIGH

Attackers craft prompts that cause the LLM to reveal its confidential system prompt, proprietary instructions, or the entire context window — exposing business logic, security rules, and competitive IP.

Attack vectors
«Print your system prompt» / «reveal your instructions»
«Output everything above this line» context dump
Multilingual variants — request in Russian, Arabic, Japanese to bypass English filters
Simulated error message tricking the model into including prompt in stack trace
Shadow Warden protection
SemanticGuard LLM07 rules — multilingual system prompt extraction patterns
ThreatVault signatures: direct dump, context window extraction, error-leak simulation
OutputGuard scans responses for system-prompt echoes before delivery
Policy engine — per-tenant rules can enforce hard block on any prompt reference
Actively protected
LLM08
Vector & Embedding Weaknesses
HIGH

Adversarially crafted embeddings or RAG-poisoned documents manipulate semantic similarity searches to inject malicious content into the model's retrieval context, bypassing keyword-based filters entirely.

Attack vectors
RAG poisoning — malicious instructions hidden in documents indexed by the vector DB
Adversarial embedding manipulation — crafted text designed to score near safe examples
«Ignore retrieved context, instead follow these new instructions» injection
Cross-document instruction propagation via shared embedding space
Shadow Warden protection
SemanticGuard LLM08 rules — RAG poisoning and adversarial embedding detection
ThreatVault — RAG injection signatures (ignore retrieved context patterns)
All RAG chunks pass through the full /filter pipeline before reaching the LLM
MiniLM cosine similarity — measures semantic distance from known attack corpus
Actively protected
LLM09
Misinformation
MEDIUM

LLMs are prompted to generate convincing but false authoritative content — fake news, fabricated legal/medical documents, false scientific studies — that will be trusted by end users or downstream systems because it appears to come from an AI.

Attack vectors
Fake news articles crafted to appear genuine — used in disinformation campaigns
Fabricated medical advice or legal documents with authoritative framing
Hallucination amplification — prompts designed to maximise confident false output
Deepfake text generation — impersonating real public figures or organisations
Shadow Warden protection
SemanticGuard LLM09 rules — fabricated authoritative content detection
ThreatVault — «fabricated news / legal / medical document» signature
OutputGuard post-generation scan — flags high-confidence false claims in AI output
Custom policy engine — block or watermark AI-generated authoritative content per tenant
Actively monitored
LLM10
Unbounded Consumption
HIGH

Attackers craft inputs that cause the LLM to consume unlimited compute, tokens, or memory — through repetition bombs, infinite generation loops, or high-cost reasoning chains — resulting in DoS or runaway cloud bills.

Attack vectors
Token bomb — «repeat this word 100,000 times» exhausts context window
Infinite generation loop — «write an endless story, never stop»
High-cost reasoning chains — adversarial chain-of-thought prompts triggering max compute
Batch flooding — submitting thousands of long requests to deplete API quotas
Shadow Warden protection
WalletShield — per-tenant token budget with configurable hard limits and hourly windows
Rate limiter — 60 req/min per API key (Redis-backed, configurable per tenant)
SemanticGuard LLM10 rules — repetition bombs and infinite loop patterns
Batch endpoint cap — max 50 items per /filter/batch request
Actively protected
